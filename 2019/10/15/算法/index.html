算法的学习

常见函数的时间复杂度

算法复杂度 1s可处理的规模
lgn 2*(100000000)
n 100000000（10的8次方）
n² 10000（10的4次方）
n³ <1000
2^n 27

反过来想：运行10^8的规模，需要消耗的时间

复杂度/n的规模 100000000（10的8次方）
lgn 27/10^8 s
n 100000000（10的8次方）
n² 10^8 s
n³ 10^16 s
2^n 巨大

总结：

时间复杂度从低到高（优->差）

O(1) < O(lgN) < O(sqrt(N)) < O(N) < O(NlgN) < O(N) < O(N*N) < O(2^N) < O(N^N)

基础排序算法性能对比：

O(N*N)：冒泡、插入、选择

O(nlgn)：jdk自带的快排(Arrays.sort())

如何写好api接口文档

一、为什么写

  1. 当完成一个API接口时，通常需要与前端人员沟通接口的细节，这时候，一个文档就能节省很多效率
  2. 项目维护中或者项目人员更迭,方便后期人员查看、维护

二、如何写

​ 功能模块标题

  1. 接口简要描述：

    • 主要说明该接口是实现什么功能的，如用户登录接口

  2. 请求URL地址：

    • http://xx.com/api/user/login

  3. HTTP请求方式：

    • GET/POST

  4. 请求头部：

  5. 请求参数：JSON或表单

参数名 类型 是否必须 说明

password str 是 密码

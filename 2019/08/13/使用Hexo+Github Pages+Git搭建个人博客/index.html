使用Hexo+Git+Github Pages搭建个人博客

一.什么是Hexo？

  1. Hexo是基于Node.js，是一个快速、简洁且高效的博客框架。
  2. Hexo 使用 Markdown解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

二.为什么要用Hexo？

优势：

  • 轻量级，无需数据库，发布环境下是静态页面 - Hexo
  • 直接在github pages平台上托管我们的博客
  • 利用MarkDown语法，可以写出简洁优质的博文

总结：

  • hexo 正常来说，不需要部署到我们的服务器上，我们的服务器上保存的，其实是基于在hexo通过markdown编写的文章，然后hexo帮我们生成静态的html页面，然后，将生成的html托管到Github pages去，可以省去服务器维护的时间和经理。

    简而言之：hexo是个静态页面生成、上传的工具。

  2. Git（自行下载好）
  3. Github账号注册
  4. Notepad++或sublime等编辑器

四.将Hexo与Github page联系起来

1.设置Git的user name和email

进入安装了hexo的文件夹空白处右键，点击Git Base Here(安装了git才会出现的)，

2.生成新的SSH Key，如果没有.ssh文件，会为你自动创建.ssh文件夹

然后系统会要你输入密码：如果你不想在使用密钥时输入口令，将其留空即可（即直接按回车）。如果你输入的话，要记得，到时候会用到

输完密码后，你可以去开始，打开你的用户目录，有一个“.ssh”文件夹，里面有“id_rsa” “id_rsa.pub” 和 “known_hosts”这三个文件。其中“id_rsa”是你的私钥文件，“id_rsa.pub”是你的公钥文件，一会要用公钥文件。

3.把上述文件id_rsa.pub用Notepad++或sublime打开，然后全选复制里面的内容

4.登录到Github，然后新建一个仓库

注意：新建仓库，项目名必须要遵守格式：账户名.github.io（即以下两个红框要一致），不然接下来会有很多麻烦，将来你的网站访问地址就是http://用户名.github.io 。并且需要勾选Initialize this repository with a README

由此可见，每一个github账户最多只能创建一个这样可以直接使用域名访问的仓库。

5.点击的settings后，点击Deploy Keys

6.点击Add deploy key，然后把刚刚复制的公钥粘贴进去,title可不填，勾选Allow write access

7.测试添加ssh是否成功。如果看到Hi后面是你的用户名，就说明本地就和Github服务端连接上了

八.如何在Hexo博客文章中插入图片

前言：

在写文章时，常常有配图说明的需求。Hexo有多种图片插入方式，这里介绍一种稳定好用的方法

网上的很多做法是通过本地上传照片来实现在Github Pages上显示照片的，这会导致一个问题，占用Github Pages空间，加载很慢，毕竟github的服务器在国外，而且不稳定，容易失效

我这里的做法是：

在阿里云那里买一个对象存储服务（OSS）价格可以接受，专门很来存放我博客里的照片的，在把图片上传到OSS后，会生成对应的url地址，将地址直接拿来引用即可。这样就可以达到稳定高效

九.hexo常用命令

hexo命令基本只需要记住下面的命令即可（左边是缩写 右边是完整的命令）：

hexo g == hexo generate —生成静态文件

hexo d == hexo deploy —部署到服务器

hexo s == hexo server— 开启本地服务

hexo n == hexo new— 新建

hexo clean —清除

hexo g -d –生成并上传

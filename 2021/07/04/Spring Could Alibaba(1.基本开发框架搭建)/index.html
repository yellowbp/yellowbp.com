Spring Cloud Alibaba（1:基本开发框架搭建）

SpringCloud Alibaba对运行环境有一定的版本要求：

  1. 64 bit JDK 1.8+
  2. Maven 3.2.x+

搭建分布式架构

先来体验一下分布式架构下，服务与服务之间的通讯是如何实现的，然后再慢慢过渡到SpringCloud

创建多Module工程

基于SpringBoot的父maven项目

  1. 通过idea自带的Spring Initializr进行创建父项目，Type选择’Maven POM’，然后一直next即可

    • 当然你也可以选择maven去创建，这两种创建方式都需要你自己手动去修改一些配置，Spring Initializr会自动帮我们继承父依赖，与自动帮我们引入一些常用的场景启动器，所以会方便一些
    • 父Maven项目一般是用来聚合子Maven项目，即管理子项目，或者说子项目一般是继承父Maven项目的依赖和配置的，所以父Maven项目不会作为一个具体项目去使用，也就是说我们不会把它打成一个jar包，Type选择POM，这样在创建父Maven项目后，不会为我们创建src、resources，只有pom.xml，但因为packing只有jar和war，所以这个地方在创建项目后需自行修改

  2. 修改pom.xml

创建子Module

  1. 右键项目，创建module，然后选择Maven创建

    • 这里不要选择spring Initializr，因为这样的话，又会在新建模块的pom.xml创建很多与父Maven相同的配置。子Maven项目时可以去继承父Maven项目的，所以我们不需要去同样创建相同配置

    • 这时候会自动帮我们继承父Maven项目

    • 按照上述步骤创建2个子Maven模块，分别为order（订单服务）、stock（库存服务）

这样一个简单的开发框架就搭建起来了。

创建rest接口，简单地实现服务的调用

让订单接口调用库存接口

  1. 要写rest接口，就需要在order和stock依赖springMvc，在各自的pom中引入web启动器即可

  2. 各自服务创建controller、application.yml和启动器

  3. 配置RestTemplate来实现服务的调用

    • springboot之前我们一般是用httpclient，现在我们可以用RestTemplate

    在order服务下新建config来配置RestTemplate

    1. 在order中的controller注入restTemplate，并调用stock服务中的接口

    2. 配置文件yml order端口8010 stock端口8011

SpringCloudAlibaba微服务架构搭建

在引入springcloudAlibaba的时候，我们必须要注意版本适配的问题，因为如果某一个组件跟框架版本不适配，很可能会出现问题。

版本依赖关系查看

下面我的项目使用的版本：

Spring Cloud Alibaba:2.2.5.RELEASE

Spring Boot :2.3.2.RELEASE

Spring Cloud:Hoxton.SR8 （这个版本也称为H版，这个版本命令是根据伦敦地铁站命名的，当然如果没有用到SpringCloud里面的组件的话，这个可以不引）

父Maven项目pom.xml修改如下

  • 这样项目相当于继承了三个版本管理，因为parent标签只能存在一个，所以使用dependencyManagement来完成继承关系
  • dependencies 和 dependencyManagement 区别
    1. dependencies即使在子项目中不写该依赖项，那么子项目仍然会从父项目中继承该依赖项（全部继承）
    2. dependencyManagement里只是声明依赖，并不实现引入，因此子项目需要显示的声明需要用的依赖。如果不在子项目中声明依赖，是不会从父项目中继承下来的；只有在子项目中写了该依赖项，并且没有指定具体版本，才会从父项目中继承该项，并且version和scope都读取自父pom;另外如果子项目中指定了版本号，那么会使用子项目中指定的jar版本。

到此springcloud Alibaba环境就已经搭建完成了，这样我们就可以在子Maven项目添加我们的微服务组件了。

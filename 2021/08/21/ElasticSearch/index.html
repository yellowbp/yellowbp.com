ElasticSearch(ES)

准备工作

  1. jDK1.8（至少）

ES安装

  1. 官方地址

  2. 自行选择下载版本（选择7之后的，因为6跟7区别较大），以下是以7.8版本的win为例

  3. Windows 版的 Elasticsearch 的安装很简单，解压即安装完毕

  4. 目录结构

    目录 含义
    bin 可执行的脚本文件
    config 配置目录
    jdk 内置JDK目录(一般不使用，而是用系统的JDK)
    lib 类库。如果系统配置 JAVA_HOME，那么使用系统默认的 JDK，如果没有配置使用自带的 JDK，一般建议使用系统配置的 JDK。
    log 日志目录
    moudules 模块目录
    plugins 插件目录

启动ES

进入 bin 文件目录，点击 elasticsearch.bat 文件启动 ES 服务

【注意】：9300 端口为 Elasticsearch 集群间组件的通信端口，9200端口为浏览器访问的 http协议 RESTful 端口（即通过9200端口来访问9300）

【测试】打开浏览器（推荐使用谷歌浏览器），输入地址：localhost:9200，测试结果

ES概念

Elasticsearch 是面向文档型数据库，一条数据在这里就是一个文档。为了可以简单理解：可以把ES理解成Mysql

概念类比：

ElasticSearch Mysql
Index（索引） Database（数据库）
Type（类型） Table（表）
Document（文档） Row（数据行）
Feild（字段） Column（列）

【注意】：这里 Types 的概念已经被逐渐弱化，高版本不会出现type的概念了。ES 是使用Json数据格式进行交互的

创建索引

对比关系型数据库，创建索引就等同于创建数据库

在 Postman 中，向 ES 服务器发 PUT 请求 ：127.0.0.1:9200/shopping

请求后，服务器返回相应

响应参数说明：

  • acknowledged：【响应结果】
  • shards_acknowledged： 【分片结果】创建索引库的分片数默认 1 片，在 7.0.0 之前的 Elasticsearch 版本中，默认 5 片
  • index： 【索引名称】

如果重复添加索引，会返回错误信息，因为 PUT 具有幂等性（即两次返回请求返回结果一样）

映射操作

有了索引库，等于有了数据库中的 database。接下来就需要建索引库(index)中的映射了，类似于数据库(database)中的表结构(table)。创建数据库表需要设置字段名称，类型，长度，约束等；索引库也一样，需要知道这个类型下有哪些字段，每个字段有哪些约束信息，这就叫做映射(mapping)

映射数据说明：

  • 字段名：任意填写，下面指定许多属性，例如：title、subtitle、images、price

  • type：类型，Elasticsearch 中支持的数据类型非常丰富，说几个关键的：

    • String 类型，又分两种：

    • keyword：不可分词，数据会作为完整字段进行匹配

    • Numerical：数值类型，分两类

      ​ 基本数据类型：long、integer、short、byte、double、float、half_float

      ​ 浮点数的高精度类型：scaled_float

    • Date：日期类型

    • Array：数组类型

    • Object：对象

  • index：是否索引，默认为 true，也就是说你不进行任何配置，所有字段都会被索引。

    true：字段会被索引，则可以用来进行搜索

    false：字段不会被索引，不能用来搜索

  • store：是否将数据进行独立存储，默认为 false

    原始的文本会存储在_source 里面，默认情况下其他提取出来的字段都不是独立存储的，是从_source 里面提取出来的。当然你也可以独立的存储某个字段，只要设置”store”: true 即可，获取独立存储的字段要比从_source 中解析快得多，但是也会占用更多的空间，所以要根据实际业务需求来设置。

  • analyzer：分词器，这里的 ik_max_word 即使用 ik 分词器,后面会有专门的章节学习
